React - 프론트엔드 자바스크립트 기반 라이브러리

명령형 - 알고리즘 기반 처리방식
React는 선언형 프로그래밍 패러다임

함수 컴포넌트는 useState Hook을 사용하여 내부 상태를 사용


1. React의 탄생
    (1) 탄생 배경
    - React는 Facebook의 개발팀이 화면 업데이트를 위해 개발하던 중 탄생헀다
    - React는 자바스크립트 기반의 라이브러리
    - 2013년에 등장한 선언형 바식으로 UI를 표현함
    - 현재는 React.19v 을 API 지원 : 항상 최신버전으로 개발 권장

    (2) 왜 새로운 기술이 필요했나
    - 기존의 기술 한계점 : 정적인 웹 구축에 초점을 맞췄다, 대규모 앱 구축에 부적합
    - Facebook의 요청 : 다양한 상호작용(사용자와의 양방향 소통))
    - 변화가 잦은 대규모 앱 필요했다

    React 핵심 목적 - 변화가 자주 일어나는 대규모 애플리케이션에 적합한
    쉽고 효율적인 라이브러리를 만들자!

2. React의 특징 3가지
    (1) 컴포넌트 기반
    - UI를 독립적이고 재사용 가능한 컴포넌트 단위로 분리할 수 있다
    (2) 가상돔 (Virtual DOM)
    - 실제 DOM을 복제하여 가상돔에서 필요한 부분만 업데이트를 진행한 후 한번에 실제 DOM에 반영하여 빠른 업데이트를 진행
    (3) 단방향 데이터 흐름
    - 부모 컴포넌트 -> 자식 컴포넌트로 데이터를 전달함
    - 자식 컴포넌트 -> 부모 컴포넌트로 데이터 전달 불가

3. React 앱 생성 - ( React 프로젝트)
    (1) Vite(비트) = React 라이브러리를 제공하는 오픈 API
    (2) 기존 CRA ( Create React App )을 대체하는 라이브러리
        CRA는 2025년 기준 deprecate
    (3) Vite 활용한 리엑트 앱 생성
        1) 개발 환경 구성 - Node.js 설치 및 확인
            node -v / npm -v
        2) React 앱 생성 : my-app 
            형식 : npm create vite@latest [프로젝트명 ( 앱이름 )]
            npm create vite@latest my-app
